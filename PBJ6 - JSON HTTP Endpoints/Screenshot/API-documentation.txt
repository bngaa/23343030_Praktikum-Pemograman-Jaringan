=== TEKNOLOGI YANG DIGUNAKAN ===

Backend:
- Node.js (JavaScript Runtime)
- Express.js (Web Framework)
- Handlebars (Template Engine)
- postman-request (HTTP Request Library)

Frontend:
- HTML5
- CSS3
- Handlebars Templates

API Eksternal:
- Mapbox Geocoding API (koordinat dari alamat)
- WeatherStack API (data cuaca real-time)

=== STRUKTUR REQUEST FLOW ===

1. User mengirim GET request ke /infocuaca?address=Jakarta
2. Express router menangkap request
3. Validasi parameter address
4. Panggil geocode() → Mapbox API menghasilkan lat/long
5. Panggil forecast() → WeatherStack API menghasilkan data cuaca
6. Format response JSON
7. Kirim response ke client

=== ERROR HANDLING ===

✓ Missing parameter: Kirim error message
✓ Invalid location: API returns error handling
✓ API connection error: Return error message
✓ Invalid route: Redirect to 404 page
✓ All responses properly formatted as JSON
